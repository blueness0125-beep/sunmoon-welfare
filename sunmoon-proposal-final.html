SECTION 01

현재 방식의 문제점

선문대학교는 병원 42개, 숙박 18개, 장례 4개 등 우수한 복지 인프라를 보유하고 있습니다. 그러나 정보 전달 방식의 한계로 인해 구성원들의 실제 이용률이 저조한 상태입니다.
📂

정보 접근의 파편화

그룹웨어 게시판 내 개별 게시물이나 76장 분량의 PDF 파일로 배포되어, 필요한 순간에 즉시 정보를 찾기가 어렵습니다. 73개 기관을 일일이 넘겨가며 확인해야 합니다.

📱

모바일 가시성 부족

이동 중 병원 예약이나 숙박 혜택 확인이 필요한 상황에서 무거운 PDF 파일을 열어보는 것은 UX(사용자 경험) 측면에서 매우 불편합니다. 확대/축소를 반복하며 정보를 찾아야 합니다.

🔄

실시간성 부재

협약 종료나 혜택 변경 시 파일 재배포가 필요하며, 구버전 데이터를 혼동할 위험이 있습니다. 관리자도 매번 PDF를 수정하고 재업로드하는 반복 작업을 수행해야 합니다.

🔒 추가 발견: 보안 취약점

현재 PDF에는 일성콘도, 하이원리조트 등의 예약 로그인 ID/비밀번호가 평문으로 노출되어 있습니다. 이 문서가 교직원 외부로 유출될 경우 법인 예약 계정이 악용될 위험이 있으며, 이는 즉시 해결이 필요한 보안 이슈입니다.

👥 학생 혜택 통합 필요 강력 권장

현재 73개 협약기관 중 17개(23.3%)는 재학생도 이용 가능하지만, PDF에서는 교직원 혜택과 학생 혜택이 슬라이드별로 혼재되어 있어 학생이 자신에게 해당되는 기관을 파악하기 어렵습니다. 외국인 유학생(1개 기관), 교육생(1개 기관) 등 세부 대상자 구분도 필요합니다. 새로운 플랫폼에서는 "학생 혜택" 원터치 필터로 즉시 분리하여 보여줍니다.

⚠️ 추후 학생지원팀과 협의를 통해 강력히 통합을 권장합니다

학생 혜택 데이터는 현재 교직원 복지 정보와 분리되지 않은 채 관리되고 있습니다. 플랫폼 구축 시 학생지원팀과 긴밀히 협의하여 대상자 구분 체계를 정립하고, 학생 전용 혜택 카테고리를 통합 운영하는 방안을 강력히 권장합니다. 이를 통해 재학생의 복지 혜택 접근성을 획기적으로 높일 수 있습니다.

SECTION 03

개발 전략 및 기술

최신 웹 기술을 적용하여 데스크톱과 모바일에서 동일한 최적의 경험을 제공합니다.

핵심 개발 방향

데스크톱(PC)을 기본 환경으로 설계하되, 100% 반응형으로 모바일·태블릿에서도 최적화된 화면을 제공합니다. 접속 단말기에 따라 레이아웃이 자동으로 변환됩니다.

반응형 웹 (Responsive) 최신 프레임워크 AES-256 암호화 SSO 통합인증 카카오맵 API AI 챗봇 (RAG)

주요 기능 가이드

🔍

스마트 검색

기관명, 혜택 내용, 주소를 동시에 검색. "라식", "할인", "천안" 등 키워드로 즉시 필터링됩니다.

📱

100% 모바일 지원

데스크톱 3단 카드 → 태블릿 2단 → 모바일 1단으로 자동 전환. 터치 최적화 44px 버튼.

📞

원터치 전화·길찾기

전화 버튼 터치 시 즉시 통화 연결. 길찾기 버튼은 카카오맵/네이버맵으로 경로 안내.

🗂️

카테고리·대상자 필터

병원(8분류)/숙박/장례/기타 카테고리 + 전체/학생/교직원 대상자 필터 조합.

🗺️

지도 API 연동

카카오맵 Geocoding으로 기관 좌표 자동 변환. 현재 위치 기준 가까운 순 정렬 제공.

🔒

민감정보 암호화

숙박 예약 ID/PW를 AES-256으로 암호화 저장. 교직원 SSO 인증 후에만 열람, 접근 기록 자동 저장.

⚙️

관리자 CMS

인사총무팀이 직접 기관 추가/수정/삭제. 엑셀 일괄 등록. 변경 이력 자동 추적.

🤖

AI 복지 어시스턴트

"치과 할인 가장 큰 곳?" 자연어로 질문하면 최적 기관을 추천하는 대화형 AI.

시안 A — 기업 복지몰

베네피아(benepia.co.kr) 레퍼런스. 좌측 사이드바에 카테고리 8종 + 대상자 필터가 고정되고, 우측에 이미지가 포함된 3단 카드 그리드가 채워지는 구조입니다. 상단에 캠퍼스 항공사진 기반 히어로 배너와 핵심 통계(73개 기관, 42개 의료 등)가 표시됩니다.

  • 정보 밀도 높음 — 한 화면에 6개 기관 동시 확인
  • 관리자 대시보드와 톤 통일 가능
  • 이미지 카드로 시각적 풍부함
  • 카드 높이가 커서 스크롤이 다소 길어짐
  • 사이드바 공간으로 카드 영역이 좁아질 수 있음

모바일 전환: 사이드바가 상단 가로 스크롤 칩으로 변환, 카드는 1단 풀 너비로 전환됩니다.

시안 B — 감성 카드형

위펀 스낵24(wefuncorp.com) 레퍼런스. 큰 그래디언트 배너 + 사진 기반 퀵 카테고리 메뉴(4종) + 사진이 포함된 3단 카드 그리드 구성입니다. 카드 상단에 카테고리별 컬러 액센트 바가 시각적 구분을 제공합니다.

  • 친근하고 감성적인 분위기 — 학생에게 특히 친화적
  • 사진 퀵 메뉴로 직관적 카테고리 탐색
  • hover 시 이미지 줌 등 인터랙션 풍부
  • 정보 밀도가 상대적으로 낮음
  • 혜택 상세를 보려면 카드 클릭 후 진입 필요

모바일 전환: 퀵 메뉴 2×2 배치, 카드 1단 풀 너비, 배너 세로 레이아웃으로 전환됩니다.

시안 C — 프리미엄 미니멀

LG라이프케어(lglifecare.com) 레퍼런스. 화이트 기반 리스트형 카드에 좌측 썸네일 이미지가 붙는 구조입니다. 한 줄에 기관명 + 혜택 + 주소 + 전화번호 + 액션 버튼이 모두 보여서 데스크톱에서 가장 효율적입니다.

  • 데스크톱 정보 효율 최고 — 스크롤 없이 6개 기관 확인
  • 깔끔하고 전문적인 인상 — 장기 운영 안정성
  • 확장성 높음 — 기능 추가 시 레이아웃 파괴 없음
  • 카드형 대비 시각적 화려함은 상대적으로 적음

모바일 전환: 리스트 카드가 세로 카드로 변환, 썸네일이 좌측에서 상단으로 이동합니다.

별첨 파일(sunmoon-design-proposals.html)을 브라우저에서 열면 3가지 시안의 실제 인터랙션(hover 효과, 반응형 전환 등)을 직접 확인할 수 있습니다. 실제 기관 사진은 협약기관 제공 이미지로 교체됩니다.
